# 03_Architecture

真正的架构设计。

包括 RFC、ADR、Prototype。这里才讨论 Kernel、Capability、Workflow、Context 等。

## 子目录

- `RFC/` — 方案提案（允许被推翻、讨论、修改）
- `ADR/` — 架构决策记录（已达成共识、经过验证）
- `Prototype/` — 实验记录（EXP）

## 纪律

- 每个 RFC / ADR 必须引用其解决的 Problem（`Relates-To: PS-XXXX`）。
- 每个架构决策都必须可追溯、可验证。
- 交叉引用规范见根 `README.md` 第 6.2 节。
